### 核心模块

| 模块 | 职责 | 关键设计 |
|------|------|---------|
| schedule | 排期日历，可用性算法 | Easy!Appointments 借鉴 + Redis 缓存 |
| order | 订单 CRUD，并发控制 | 乐观锁 + 幂等 key |
| wechat | 微信登录，消息推送 | code2session + 订阅消息 |
| config | 全局配置管理 | 每日上限 / 休息日 / 名额覆盖 |
| auth | 管理员鉴权 | JWT + Guard |
| common | 公共能力 | Redis / 幂等 / 限流 |

### 数据模型

```
schedules ──1:N──> orders ──N:1──> users (openid)
    │
config (key-value)
```

### 关键约束

- 每日名额 = max_slots - booked_slots（不含已取消）
- 同一手机号同日期仅一单（联合唯一索引）
- 乐观锁 version 防超卖
- 幂等 key（UUID v4）防重复提交
