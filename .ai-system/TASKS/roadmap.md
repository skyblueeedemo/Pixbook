```

─────────┼─────────┼────────────────────────────────────
Week 2   │         │ 配置 API + 排期日历 API
Week 3   │ Phase 1 │ 订单 API + 微信登录 + 鉴权 + 并发测试
─────────┼─────────┼────────────────────────────────────
Week 4   │         │ 小程序基础 + 日历组件
Week 5   │ Phase 2 │ 预约表单 + 成功页 + 小程序特性适配
─────────┼─────────┼────────────────────────────────────
Week 6   │         │ 管理后台：排期 + 订单列表

─────────┼─────────┼────────────────────────────────────
Week 8   │ Phase 4 │ 集成测试 + 性能优化 + 上线部署
─────────┼─────────┼────────────────────────────────────
Week 9+  │ Phase 5 │ 迭代增强（按需）
```

## Phase 0 · 项目初始化（第 1 周）

**目标：** 搭建工程骨架，统一开发环境

**交付物：**
- 可运行的空项目骨架（含 uni-app 小程序工程）
- `docker-compose.yml`（本地开发环境）
- Prisma schema + migration 文件
- 微信开发者工具可预览的空白小程序
- README 包含本地启动步骤

## Phase 1 · 核心后端（第 2–3 周）

**目标：** 全部 P0 级 API，并发安全

**模块：**
1. 配置服务（GET/PUT config）
2. 排期日历 API（可用性算法 + Redis 缓存）
3. 订单提交 API（乐观锁 + 幂等）
4. 微信登录模块（code2session）
5. 管理员鉴权（JWT）

## Phase 2 · 客户端小程序（第 4–5 周）

**目标：** uni-app 实现完整客户预约流程

**模块：**
1. 日历组件（BookingCalendar / DayCell / CalendarLegend）
2. 预约表单（BookingForm + 5 字段校验）
3. 预约成功页（订单凭证）
4. 订单查询页
5. 微信特性（一键手机号、订阅消息）

## Phase 3 · 管理后台（第 6–7 周）

**目标：** 管理员全部操作界面

**模块：**
1. 排期管理（月历视图 / 设置上限 / 休息日）
2. 订单列表（筛选 + 分页）
3. 订单详情 + 状态流转

## Phase 4 · 集成测试 + 上线（第 8 周）

**目标：** 全链路测试，性能达标，准备上线

**交付物：**
- 测试报告（功能 + 并发 + 安全）
- 小程序审核提交
- 上线部署文档

## Phase 5 · 迭代增强（第 9 周起，可选）

- 微信订阅消息推送（P1）
- 数据统计图表（P2）
- 候补队列（V2.0）
- 微信/支付宝支付（V1.2）
- 多管理员支持（V2.0）
